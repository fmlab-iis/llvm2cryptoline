The IR is extracted by the following compiling command:

llvm-7.0.0/bin/clang -S -emit-llvm -Ibin/.build/reference/libXKCP.a/ -Ilib/high/Keccak/ -Ilib/high/Keccak/FIPS202/ -Ilib/high/Keccak/SP800-185/ -Ilib/high/common/ -Ilib/high/Keccak/PRG/ -Ilib/high/Ketje/ -Ilib/high/Keyak/ -Ilib/high/KangarooTwelve/ -Ilib/high/Kravatte/ -Ilib/high/Xoofff/ -Ilib/high/Xoodyak/ -Ilib/common/ -Ilib/low/KeccakP-200/ref/ -Ilib/low/KeccakP-400/ref/ -Ilib/low/KeccakP-800/ref/ -Ilib/low/KeccakP-1600/ref-64bits/ -Ilib/low/KeccakP-1600-times2/fallback-on1/ -Ilib/low/common/ -Ilib/low/KeccakP-1600-times4/fallback-on1/ -Ilib/low/KeccakP-1600-times8/fallback-on1/ -Ilib/low/Xoodoo/ -Ilib/low/Xoodoo/ref/ -Ilib/low/Xoodoo-times4/fallback-on1/ -Ilib/low/Xoodoo-times8/fallback-on1/ -Ilib/low/Xoodoo-times16/fallback-on1/ -Ilib/low/Ketje/SnP-compliant/ -Itests/UnitTests/  -fomit-frame-pointer -O2 -g0 -O -c lib/low/KeccakP-1600/ref-64bits/KeccakP-1600-reference.c -o bin/KeccakP-1600-reference.ll

That is the command from "make reference/libXKCP.a" excluding the option "-DKeccakReference". "-DKeccakReference" will put the computation of KeccakRhoOffsets into a separate function (instead of pre-computed), which makes KeccakP1600Round contains many branches.
